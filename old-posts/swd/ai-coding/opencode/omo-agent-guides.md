### Agent 分类

```
Primary Agents（主 agent）: 4个 - 负责任务编排和执行
├── Sisyphus（西西弗斯）- 主编排
├── Hephaestus（赫菲斯托斯）- 深度实现
├── Atlas（阿特拉斯）- 会话编排
└── Prometheus（普罗米修斯）- 战略规划

Sub Agents（子 agent/专家）: 7个 - 提供专业领域支持
├── Metis（墨提斯）- 预规划分析
├── Momus（摩摩斯）- 计划验证
├── Oracle（神谕）- 设计/调试顾问
├── Librarian（莱布瑞恩）- 文档/代码搜索
├── Explore（艾克斯）- 快速代码探索
├── Multimodal Looker（莫达尔）- 图像/PDF分析
└── Sisyphus-Junior（小西西弗斯）- 子任务执行
```

### 核心设计哲学

- **专业化分工**: 每个 agent 都有明确的职责边界
- **协作而非竞争**: agent 之间相互配合，形成完整工作流
- **推石头精神**: 任务不完成绝不停止（Sisyphus 核心机制）

> ⚠️ **重要提示**: Hephaestus **没有 fallback 模型**。如果主模型不可用，任务会直接失败。这是设计决策，确保执行的一致性和可预测性。

```

```

**特点**:

- 🔥 "正当工匠"——不抄近路，methodical 执行
- 🎨 先探索研究，再精准实现
- 🎯 给你一个目标，他自己搞定一切

**读写权限**: ✅ 全权限（但不能创建任务，只能执行）

**使用场景**:

```bash

# 深度重构
opencode "将这个单体应用拆分为微服务"
```

**与西西弗斯的区别**:

- **西西弗斯**: 我给你规划，让团队来做
- **赫菲斯托斯**: 我自己研究，我自己做，直到完成

## 使用场景与示例

### 快速开发功能（最常用）

**需求**: 实现一个完整的用户认证系统

**推荐入口**: Hephaestus（赫菲斯托斯）

```bash
opencode "给我实现一个完整的用户认证系统，包括注册、登录、密码重置"
```

**执行过程**:

1. 赫菲斯托斯启动预研，调用多个艾克斯并行搜索
2. 莱布瑞恩研究 JWT 最佳实践
3. 神谕提供安全建议
4. 赫菲斯托斯独立完成实现
5. 自动测试验证

**预计耗时**: 10-30分钟（取决于复杂度）

### 复杂项目重构

**需求**: 重构整个前端架构

**推荐入口**: Sisyphus（西西弗斯）+ `ulw` 关键词

```bash
opencode "ulw 重构整个前端架构，从 Class 组件迁移到 Hooks"
```

**执行过程**:

1. 西西弗斯分析整体影响
2. 协调多个 agent 并行工作
3. 小西西弗斯批量处理不同模块
4. 持续推动直到全部完成

**预计耗时**: 1-数小时（ultrawork 模式会持续执行）

### 先规划再执行

**需求**: 开发一个新的电商平台

**推荐入口**: Prometheus（普罗米修斯）→ Hephaestus（赫菲斯托斯）

**步骤 1**: 规划阶段

```bash
opencode "我要做一个电商网站，帮我规划一下架构和技术选型"
# agent: 普罗米修斯
```

**步骤 2**: 执行阶段（根据生成的计划）

```bash
opencode "按照刚才的计划，开始实现用户模块"
# agent: 赫菲斯托斯
```

### 快速定位代码

**需求**: 找到特定功能的实现位置

**推荐入口**: Explore（艾克斯）

```bash
opencode "找到认证中间件的实现代码"
# agent: 艾克斯（最快最经济）
```

**预计耗时**: 几秒钟

### Debug 问题

**需求**: API 响应很慢，需要诊断

**推荐入口**: Oracle（神谕）

```bash
opencode "这个API响应很慢，帮我找出原因并优化"
# agent: 神谕 → 赫菲斯托斯
```

**执行过程**:

1. 神谕诊断问题根因
2. 艾克斯搜索相关代码
3. 莱布瑞恩查性能优化文档
4. 赫菲斯托斯实施修复

### 研究技术方案

**需求**: 了解新技术的工作原理

**推荐入口**: Librarian（莱布瑞恩）

```bash
opencode "React Server Components 的工作原理是什么？"
# agent: 莱布瑞恩
```

**预计耗时**: 1-2分钟
