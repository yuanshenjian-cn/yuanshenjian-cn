# investment-daily-briefing

`investment-daily-briefing` 是面向博客的投资每日简报 workflow skill，用于查询、起草或发布“投资每日简报”。

它的三个模式是：

- 查询：只返回近 24 小时确认动态、未来重点观察和待核验线索
- 成稿：生成可审阅的 Markdown 草稿，并附带内部审核摘要
- 发布：在审核通过后写入博客、更新索引并执行 git 发布

## 默认原则

- 默认模式：查询
- 默认节奏：北京时间早报
- 默认结构：`近 24 小时确认动态` + `未来重点观察`
- 默认采集窗口：执行时点往前 24 小时
- 字数统计：不包含 `## 来源` 章节
- `高共识预期`：默认关闭
- `传闻观察`：默认关闭，且不进入公开发布稿
- 连续发布时默认对照上一期做去重，无新增增量的动态不重复进入公开稿
- `config/blocked-companies.json` 中的黑名单企业相关动态一律不采集、不入稿、不进入待核验线索
- 每期 `tags` 除 `投资每日简报` 外，还应补充 2~5 个本期关键公司 / 主题 / 已官宣重点事件标签
- 红线：不提供买卖建议，不荐股，不写目标价，不写仓位建议

## 常见触发方式

```text
今天投资市场有哪些已确认的重要动态？
```

```text
帮我起草今天的投资每日简报，但不要发布
```

```text
生成并发布今天的投资每日简报
```
