# investment-briefing

`investment-briefing` 是面向博客的投资简报 workflow skill，用于查询、起草或发布“投资简报”。

它的三个模式是：

- 查询：只返回近 24 小时确认动态、未来重点观察和待核验线索
- 成稿：生成可审阅的 Markdown 草稿，并附带内部审核摘要与 reviewer 复审结论
- 发布：在审核通过后写入博客、更新索引并执行 git 发布

## 默认原则

- 默认模式：查询
- 默认节奏：按期发布
- 默认结构：`近 24 小时确认动态` + `未来重点观察`
- 默认采集窗口：执行时点往前 24 小时
- 字数统计：不包含 `## 来源` 章节
- 正文目标：短版 **1100~1399** 个中文汉字；正常版 **1400~1700** 个中文汉字
- `高共识预期`：默认关闭
- `传闻观察`：默认关闭，且不进入公开发布稿
- 连续发布时默认对照最近 5 期做去重；若已发布历史不足 5 期，则按已有全部历史检查。无新增增量的动态不重复进入公开稿
- `config/blocked-companies.json` 中的黑名单企业相关动态一律不采集、不入稿、不进入待核验线索
- 自审完成后，必须再调用项目级 `investment-briefing-reviewer` 做第二道阻断式复审
- 每期 `tags` 除 `投资简报` 外，还应补充 2~5 个本期关键公司 / 主题 / 已官宣重点事件标签
- 生成前的取舍说明、排序理由、排除理由不得写进公开稿正文，只能保留在内部审核摘要
- 开篇和章节正文都要短句、短段落、多段落；单段目标控制在 80~120 个中文汉字内
- 每个自然段只承载一个意思，不把多个公司、日期、指标和观察判断塞进同一长段
- `近 24 小时确认动态` 和 `未来重点观察` 下优先用多个短 `###` 子标题拆分主题，子标题应是短提炼，不要把完整长句搬成标题
- 正文如出现 `日期 + 星期` 组合，必须逐条核对日期与星期是否真实对应；拿不准时只写日期，不要主观补星期
- 红线：不提供买卖建议，不荐股，不写目标价，不写仓位建议

## 常见触发方式

```text
今天投资市场有哪些已确认的重要动态？
```

```text
帮我起草今天的投资简报，但不要发布
```

```text
生成并发布今天的投资简报
```

## 二次复审

当前工程里已经有项目级 `investment-briefing-reviewer`。

skill 在成稿和发布链路里，都会先完成自己的内部审核摘要，再把草稿交给 `investment-briefing-reviewer` 做第二道阻断式复审。
