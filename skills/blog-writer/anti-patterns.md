# 写作禁止清单

> 每次写作前必须读取本文件，并在写作全程严格遵守。这些是通用底线，适用于所有文章类型。

---

## 格式禁止

- **子标题不能有序号**：禁止 `## 一、` `## 二、` `### 2.1` `### 1.1` 等一切序号前缀
- **禁止章节间分割线**：不能在 `##` 级章节之间插入 `---` 分割线；文章层次由标题本身承担
- **禁止列表替代分析**：不能用"首先…其次…再次…最后"结构替代真正的论证段落
- **禁止一级标题**：正文不能出现 `#`，文章标题只放在 frontmatter 的 `title`
- **禁止 MDX/JSX**：文章文件只能是 `.md`，正文不要出现 JSX 组件或 `className`

---

## 词语禁止

以下词语/短语禁止出现：

| 禁止 | 可替代为 |
|------|---------|
| 总的来说 | 直接写结论 |
| 综上所述 / 综上 | 直接写结论 |
| 值得一提的是 | 直接陈述 |
| 值得注意的是 | 直接陈述 |
| 深入探讨 / 全面分析 / 深度解析 | 具体说做了什么 |
| 在当今 XX 背景下 | 直接进入主题 |
| 至关重要 / 尤为重要 / 不可或缺 | 说清楚为什么重要 |
| 不难发现 / 由此可见 | 直接陈述结论 |
| 可以说 / 可谓 | 直接陈述 |
| 毋庸置疑 | 直接陈述 |
| 本文将 / 接下来我们将 | 直接进入正文 |
| 从多个维度 / 多维度分析 | 说清楚具体比较什么 |
| 赋能 / 抓手 / 闭环 / 底层逻辑 | 换成具体动作或因果 |
| 重新定义 / 颠覆式 / 划时代 | 除非有充分事实支撑，否则不用 |

---

## 开头/结尾禁止

- **禁止套路式开头**：不能用"首先介绍背景，然后分析问题，最后给出建议"的铺垫结构
- **禁止客套结尾**：不能出现"希望对你有所帮助"、"欢迎留言讨论"、"感谢阅读"等客套话
- **禁止空洞总结**：结尾不能只是重复前面说过的内容
- **禁止标题式总结**：避免用 `## 总结`、`## 写在最后` 承接空泛收尾；如果确实需要结尾标题，标题本身要带判断

---

## AI 味禁止

- 禁止把文章写成平均分配篇幅的模板综述：每个部分都差不多长，但没有重点。
- 禁止连续使用“不是 A，而是 B”“既要 A，也要 B”“真正的 X 是 Y”这类排比句式。
- 禁止用抽象名词堆叠替代解释，例如“提升效率、降低成本、增强体验”但不说明具体怎么发生。
- 禁止伪具体：没有来源的数字、没有场景的人群画像、没有对象的“很多人认为”。
- 禁止过度中立：观点评论和经验复盘必须有清晰立场，不能两边都说完却不给判断。
