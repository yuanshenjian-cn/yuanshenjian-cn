4 月 8 日，OpenAI 在 Agent 基础设施和法律诉讼两个方向同时推进。Agents SDK 获得重要安全升级，同时公司正式对 Elon Musk 提起反垄断投诉。

## 速览

- OpenAI 升级 Agents SDK，新增原生沙盒执行和可配置内存。
- SDK 设计明确假设提示注入和数据外泄尝试必然发生。
- OpenAI 要求加州和特拉华州检察长调查 Elon Musk 的反竞争行为。
- 此举升级了与 Musk 之间的法律战，涉及 OpenAI 重组和 xAI 竞争。

## 重点动态

### OpenAI

OpenAI 在 4 月 8 日发布 Agents SDK 重大升级。新版本新增原生沙盒执行环境，允许 Agent 在隔离环境中安全运行代码。可配置内存功能使开发者能够控制 Agent 的上下文保留策略。安全架构设计上明确假设提示注入和数据外泄尝试必然发生，因此采用了更强的安全隔离措施。

同日，OpenAI 向加州和特拉华州检察长提交请求，要求调查 Elon Musk 涉嫌的反竞争行为。这一法律行动升级了双方之间持续数月的争端，涉及 OpenAI 从非营利转向营利结构的重组计划，以及 Musk 创立的 xAI 与 OpenAI 之间的直接竞争。

## 为什么值得关注

Agents SDK 的安全升级反映了 AI 行业对 Agent 风险认识的深化。当 Agent 获得代码执行和网络访问能力时，提示注入不再只是"越狱"问题，而是可能导致实际损害的安全威胁。OpenAI 的"假设攻击必然发生"的设计哲学代表了安全架构的重要转变——从"尽力防止攻击"转向"在攻击必然发生的前提下最小化损害"。这种"零信任"思维在网络安全领域已经成熟，但在 AI Agent 架构中的应用仍处于早期阶段。对于正在构建 Agent 应用的开发者来说，这意味着需要在设计阶段就将安全隔离作为核心考量，而不是事后补丁。原生沙盒执行环境尤其值得关注，它为 Agent 提供了一个受限的执行空间，即使 Agent 被恶意提示控制，其造成的破坏也会被限制在沙盒范围内。

OpenAI 对 Musk 的法律行动则揭示了 AI 产业竞争的另一个维度。当创始人之间的个人恩怨与商业竞争交织时，法律诉讼可能成为竞争策略的一部分。对于投资者和合作伙伴来说，这种不确定性是需要考虑的风险因素。更深一层看，这场争端的核心是 OpenAI 的重组问题——从非营利转向营利结构需要获得监管批准，而 Musk 作为 OpenAI 的联合创始人之一，其反对声音可能直接影响重组进程。xAI 与 OpenAI 在模型能力、人才招聘和企业客户方面的直接竞争，使得这场法律战不仅是个人恩怨，更关乎数百亿美元的市场格局。对于 AI 行业的观察者而言，这一案例也提出了一个重要问题：当 AI 公司的使命陈述与商业现实发生冲突时，应该如何平衡？
