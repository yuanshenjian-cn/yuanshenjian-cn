> 当各家模型还在比拼"谁更聪明"时，MiMo V2.5 Pro 选择了一条更务实的路径：用更少的 token 做更多的事，然后把价格压到同行难以跟进的水平。

2026 年 4 月 26 日，小米在 Hugging Face 上开源了 MiMo V2.5 Pro 的完整权重；次日，官方博客正式发布。这不是小米第一次做大模型——MiMo 系列从去年开始已经迭代了多个版本——但 V2.5 Pro 的发布标志着一个明确的信号：小米不再只是"参与"大模型竞赛，而是在 Agent 这条细分赛道上试图建立差异化优势。

## 核心规格：1T 参数 MoE，但激活只有 42B

MiMo V2.5 Pro 的核心架构选择是 **Mixture-of-Experts（MoE，混合专家模型）**。总参数量达到 1.02T，但每次前向传播只激活 42B 参数。这意味着它在保持理论上接近万亿参数模型表达能力的同时，实际推理成本控制在 42B 级别。

| 规格 | MiMo V2.5 Pro |
|------|--------------|
| 总参数 | 1.02T |
| 激活参数 | 42B |
| 层数 | 70（1 dense + 69 MoE）|
| 注意力头 | 128（GQA，8 KV heads）|
| 路由专家数 | 384（每 token 激活 8 个）|
| 上下文窗口 | 1M tokens |
| 训练数据 | 27T tokens（原生 32K 序列长度）|
| 精度 | FP8 (E4M3) Mixed |
| 许可证 | MIT（完全开源）|

<small>*数据来源：小米官方模型页面，2026-04-27。</small>

模型提供两个变体：Base 版支持 256K 上下文，Pro 版支持 1M 上下文。两者权重、tokenizer 和完整模型卡片均已发布在 Hugging Face 上，采用 **MIT License**——这意味着商业用途几乎不受限制。

## Agent 效率：用 40-60% 更少的 token 完成同等任务

MiMo V2.5 Pro 最值得关注的数据点不是某个单一的 SOTA 分数，而是 **token 效率**。

在 **ClawEval**（综合性 Agent 能力评测基准）上，MiMo V2.5 Pro 取得了 **64% Pass^3** 的成绩。这个分数本身已经处于第一梯队，但真正关键的是它达成这个成绩的方式：每次任务轨迹平均只消耗约 70K tokens，而 Claude Opus 4.6（小米官方博客引用的对比版本）、Gemini 3.1 Pro 和 GPT-5.4 在同等复杂度的任务中通常需要多消耗 40% 到 60% 的 token。

这个差异在实际使用中的含义是直接的。以各模型公开 API 定价对比：

| 模型 | 输入定价（per MTok）| 输出定价（per MTok）|
|------|-------------------|-------------------|
| MiMo V2.5 Pro | **$1** | **$3** |
| Claude Opus 4.7 | $5 | $25 |
| GPT-5.4 | $2.50 | $15 |
| Gemini 3.1 Pro | $2 | $12 |

<small>*MiMo 定价来自 Xiaomi MiMo API Platform 官方定价（≤256K 上下文）；竞品定价来自各厂商官方公告。</small>

MiMo V2.5 Pro 的定价本身就比所有主流旗舰低一个数量级，如果再叠加 40-60% 的 token 节省，实际单次 Agent 任务的成本可能只有 Claude Opus 4.7 的 **1/10 到 1/20**。

## 基准表现：Agent 和编程是第一梯队，数学有惊喜

| SWE-bench Verified | **78.9%** | Opus 4.7: 87.6% |

| GPQA Diamond | **66.7%** | Opus 4.7: 94.2% / GPT-5.4: 92.8% |
| MMLU-Pro | **68.5%** | Gemini 3.1 Pro: 73.5% |
| GSM8K（数学）| **99.6%** | — |
| MATH（数学）| **86.2%** | — |
| AIME 24&25 | **37.3%** | — |
| HLE | **48*** | — |

<small>*数据来源：Hugging Face 模型卡片 XiaomiMiMo/MiMo-V2.5-Pro，2026-04-28。Opus 4.7 数据来自公开评测报道。HLE 带星号表示测试条件可能有特殊说明。</small>

**编程和 Agent 能力**处于第一梯队。SWE-bench Pro 57.2% 虽然落后 Opus 4.7 约 7 个百分点，但已经超越了许多主流旗舰的初版成绩。Terminalbench 2.0 的 68.4% 是一个亮眼的数据——这个基准测试的是模型在终端环境中的工具调用和推理能力，与 Agent 场景高度相关。

**数学能力超出预期**。GSM8K 99.6% 和 MATH 86.2% 的成绩在开源模型中属于顶尖水平。这很可能受益于三阶段后训练中的"领域专项训练"——对数学推理做了独立的强化学习优化。

**通用推理（GPQA Diamond 66.7%）与顶级旗舰仍有差距**。Opus 4.7 的 94.2% 和 GPT-5.4 的 92.8% 明显更高，但这是架构选择的结果：MiMo V2.5 Pro 将优化重心放在了 Agent 效率和编程能力上，而非追求全科满分。

## 长程任务：一千次工具调用后依然稳定

小米在官方发布中展示了三个长程任务案例，这些案例的共同点是：任务持续时间长、工具调用次数极多、对错误恢复能力要求极高。

**SysY 编译器实现**：在 4.3 小时内完成 672 次工具调用，最终 233/233 测试用例全部通过。SysY 是一种教学用编译器语言，实现一个完整编译器通常需要数周的人工开发时间。

**视频编辑器开发**：在 11.5 小时内完成 1,868 次工具调用，输出 8,192 行可用代码。这个任务的特殊之处在于涉及大量跨文件依赖和 UI 交互逻辑，模型需要在长时间推理中保持对整体架构的记忆。

**模拟 EDA 设计**（FVF-LDO 电路）：在六个评估指标上全部达标，其中关键性能指标提升了一个数量级。EDA（电子设计自动化）是芯片设计领域的核心工具链，这个案例表明 MiMo V2.5 Pro 已经能处理相当专业的工程任务。

这三个案例的共同点在于：它们都不是"一次问答"能解决的问题，而是需要模型在 **数百到数千次工具调用** 中持续保持目标一致性、错误恢复能力和上下文利用效率。

在长上下文专项评测 **GraphWalks**（OpenAI 提出的长程依赖追踪基准）上，MiMo V2.5 Pro 在 1M tokens 输入长度下仍保持 0.37（BFS）/ 0.62（Parents）的得分。作为对比，上一代 MiMo V2 Pro 在 1M 时完全崩溃（0.00）。这意味着 1M 上下文不是通过简单外推"凑"出来的数字，而是模型在架构层面真正具备的长程记忆能力。

## 技术架构：Hybrid-Attention 和 MTP

MiMo V2.5 Pro 的架构创新主要体现在两个层面。

**Hybrid-Attention 机制**：模型采用了 Local Sliding Window Attention（SWA）和 Global Attention（GA）以 **6:1 的比例交错** 的设计，滑动窗口大小为 128 tokens。这个设计的直觉很简单——大部分 token 之间的注意力不需要全局计算，只有关键位置需要全局视野。6:1 的比例意味着大约 85% 的注意力计算被限制在局部窗口内，这直接降低了长序列推理的内存和计算开销。

**Multi-Token Prediction（MTP）**：MiMo V2.5 Pro 引入了一个轻量级的 MTP 模块，使用 dense FFN（全连接前馈网络）而非完整的 attention 层来预测后续多个 token。官方宣称这可以将 **输出吞吐量提升三倍**。对于 Agent 场景来说，这意味着模型在生成工具调用参数、代码片段或长文本回复时，响应延迟显著降低。

训练方面，模型采用了三阶段后训练策略：

1. **监督微调（SFT）**：建立基础指令遵循能力
2. **领域专项强化学习**：针对数学、安全、Agent 工具使用等领域，分别通过独立的强化学习进行深度优化
3. **Multi-Teacher On-Policy Distillation（MOPD）**：多教师在线策略蒸馏，从多个强模型中吸收最优策略

第三阶段尤为关键。MOPD 不是简单的知识蒸馏，而是让模型在实际交互过程中学习多个"教师模型"的决策模式——这解释了为什么 MiMo V2.5 Pro 在长程 Agent 任务中表现出较强的错误恢复能力。

## 竞争定位：性价比壁垒已经形成

将 MiMo V2.5 Pro 放入当前模型格局中看，它的位置很清晰。

**编程 Agent 能力**：MiMo Coding Bench（小米内部编程评测）上的成绩显示它正在"缩小与 Opus 4.6 的差距"——这是官方博客引用的对比版本。在独立的 SWE-bench Pro 上，MiMo V2.5 Pro 57.2% 与当前最强 Opus 4.7 的 64.3% 相差约 7 个百分点，已具备竞争力。

**通用推理能力**：ClawEval 64% Pass^3 表明它在通用 Agent 任务上具备竞争力，但尚未达到 Opus 4.7 或 GPT-5.4 的顶尖水平。

**成本效率**：这是 MiMo V2.5 Pro 最突出的差异化优势。$1/$3 per MTok 的定价加上 40-60% 的 token 节省，让它在成本敏感的大规模 Agent 部署场景中几乎没有对手。

## 局限与边界

在欢呼之前，需要明确几个限制。

**通用推理与顶尖旗舰仍有差距**：GPQA Diamond 66.7% 虽然已经是不错的成绩，但与 Opus 4.7（94.2%）和 GPT-5.4（92.8%）相比仍有明显距离。MMLU-Pro 68.5% 也低于 Gemini 3.1 Pro 的 73.5%。如果你的核心场景是科研级别的科学推导或需要顶尖通用知识储备的问答，MiMo V2.5 Pro 可能不是首选——但它在数学（MATH 86.2%）和编程上的优势已经足以覆盖大多数工程场景。

**生态成熟度**：Claude Code、OpenAI Assistants API、Google Vertex AI 都有成熟的开发者工具链和企业级服务。MiMo V2.5 Pro 目前可通过 Xiaomi MiMo API Platform（platform.xiaomimimo.com）获取官方 API 接入，同时提供 Token Plan 订阅套餐；模型已在 Hugging Face 和 ModelScope 开源完整权重，但自建部署需要大规模 GPU 集群支持（官方 SGLang 示例需 32 张 GPU），基础设施门槛较高。

**采样参数敏感**：Hugging Face 模型卡片特别提醒，本地部署时推荐 `temperature=1.0`、`top_p=0.95`。这意味着模型对采样参数较为敏感，过低的 temperature 可能导致输出质量下降——这与一些经过大量 RLHF 对齐的模型不同，后者通常在更低 temperature 下也能保持稳定性。实际部署时需要针对具体场景做参数调优。

**社区验证尚浅**：模型开源仅两天，除官方提供的案例外，社区的大规模独立评测结果还很少。ClawEval 64% Pass^3 的数据来自官方发布，第三方复现结果会是更重要的参考。

## 适合谁，不适合谁

基于目前可验证的信息，MiMo V2.5 Pro 最适合以下场景：

- **大规模 Agent 流水线部署**：如果你需要运行数百甚至数千个并行的 Agent 任务，token 成本和效率的优势会指数级放大
- **开源自托管需求**：需要完全控制模型权重、担心数据隐私、或在特定领域做持续微调的团队
- **编程辅助工具开发**：MiMo Coding Bench 上的表现表明它在代码生成和工程任务上有扎实的能力，配合开源许可证可以自由集成到 IDE 插件或 CI/CD 流程中
- **预算受限的初创团队**：用 1/10 的成本获得接近旗舰的体验，这对早期产品验证非常有价值
- **Base 版（256K 上下文）与 Pro 版基础能力一致，但推理内存占用更低、部署成本更省。如果 256K 上下文已能满足需求，Base 版是更经济的选择

不太适合的场景：

- **需要顶尖通用推理的科研任务**：如果 GPQA Diamond 或 AIME 是你的核心场景，建议等待第三方评测数据
- **对延迟极度敏感的实时交互**：虽然 MTP 提升了吞吐量，但 MoE 架构的激活路由本身会带来额外延迟，语音实时对话等场景可能不是最优选择
- **依赖成熟企业工具链**：如果你需要细粒度的权限管理、审计日志、SLA 保障等企业级服务，OpenAI 或 Anthropic 的托管方案仍然更成熟

## 小米的 AI 战略正在变得清晰

MiMo V2.5 Pro 的发布不是一次孤立的产品更新，而是小米 AI 战略的一次明确表态。

**完全开源 + 激进定价**，这是一条与 OpenAI、Anthropic 截然不同的路径。后两者靠闭源模型和订阅/API 收入建立商业模式，而小米选择了"用开源建立生态影响力，用硬件和云服务变现"的策略——这与其在手机、IoT 领域的打法高度一致。

这条路径的风险和回报都很清晰。风险在于：开源模型很难直接从模型本身获得收入，需要靠生态规模摊薄成本。回报在于：如果 MiMo 系列能在开发者社区中建立起"高性价比 Agent 模型"的心智定位，它有可能成为继 DeepSeek 之后又一个靠开源颠覆定价规则的玩家。

DeepSeek 已经证明了中国团队可以在开源模型上做出世界一流的工程能力。MiMo V2.5 Pro 则进一步证明，这种能力正在扩散——小米这样以硬件见长的公司，也能在 AI 基础设施层面推出有竞争力的产品。

真正值得关注的是：当模型本身的差距被压缩到个位数百分比，**成本效率**和**部署自由度**会不会成为比"谁更聪明"更重要的选型标准？MiMo V2.5 Pro 给出的答案是肯定的。
