# User Input 目录

这个目录用于存放用户输入的素材文件。

## 用途

当你有现成的素材文件时，可以直接放在这个目录，然后在写作时引用。

## 支持的文件类型

- `.md` - Markdown 格式（推荐）
- `.txt` - 纯文本格式
- 其他文本格式

## 文件命名建议

使用有意义的文件名：
- `微服务架构升级经历.md`
- `产品经理的思考笔记.md`
- `Vue3学习总结.txt`

## 推荐的文件格式（Markdown）

为了更好地组织内容，建议在 Markdown 文件中使用以下结构：

```markdown
# 文章主题或标题

## 我提供的素材

这里放置你提供的原始素材：
- 个人经历、真实案例
- 数据、感想、笔记
- 引用、参考资料
- 任何相关的原始信息

## 需要你额外帮我做的事情

这里列出你需要 AI 协助的事项：
- 查询某些数据或事实
- 补充背景知识或案例
- 完善论点或润色文字
- 其他任何需求
```

### 示例

```markdown
# 投资理财 | 股市热度起来了

## 我提供的素材

最近 A 股成交量维持在 2~3 万亿，创了新高。
有几个亲戚跑来问我推荐基金，他们平时不关注股市...

## 需要你额外帮我做的事情

1. 查询 A 股最近创历史新高的具体日期和成交额
2. 补充几个典型的"牛市亏钱"案例
3. 推荐 2-3 本适合新手的投资理财书籍
```

## 如何使用

在调用 blog-writer skill 时，说明要从 user-input 目录读取素材：

```
请从 user-input 目录读取 "微服务架构升级经历.md" 作为素材
```

或简写为：

```
素材：user-input/微服务升级经历.md
```

## 注意事项

- 素材文件应该包含你的真实经历、感想、笔记等
- 使用 `## 我提供的素材` 和 `## 需要你额外帮我做的事情` 分隔内容更清晰
- AI 会优先使用素材部分的内容，同时完成需求部分的任务
- 文件内容可以很随意，不要求格式完美
