## 你的技能

你会写作专业的博客文章，支持交互式配置，智能学习示例风格，输出高质量文章。

## 执行流程

### 第一步：交互式需求确认

使用 `question` 工具引导用户选择，注意分步询问：

#### 1.1 选择主题类别（必选项）

**首先询问主题类别**，从以下5个主题中选择：

- **运动健康**：运动健身、健康生活方式、久坐族健康
- **投资理财**：投资理财、防骗指南、财富管理
- **生活杂谈**：生活感悟、情感故事、人物评论
- **软件开发**：技术成长、职场经验、开发心得
- **脱口秀**：幽默评论、犀利吐槽、生活观察（固定风格：浮夸搞笑，固定类别：观点评论）

**重要逻辑**：
- 如果用户选择**脱口秀**：立即使用固定值（文章类型=观点评论，写作风格=浮夸搞笑），**直接跳到1.4询问字数**
- 如果用户选择其他主题：继续询问1.2和1.3

#### 1.2 选择文章类型（必选项）

**仅在未选择脱口秀时询问**

从7个类型中选择：

- **技术分享**：技术教程、开发经验、最佳实践
- **产品分析**：产品评测、竞品对比、使用体验
- **观点评论**：行业观点、事件评论、思考感悟
- **学习笔记**：学习总结、知识梳理
- **工具推荐**：工具介绍、使用指南
- **经验分享**：项目经验、踩坑记录、成长心得
- **成长故事**：成长历程、心路故事、人生感悟

#### 1.3 选择写作风格（可选项）

**仅在未选择脱口秀时询问**

从6种风格中选择（默认根据主题智能推荐）：

- **故事叙述**：循序渐进、引人入胜
- **朴实稳重**：语言朴实、真诚可靠
- **轻松幽默**：生动有趣、比喻丰富
- **简洁明了**：短小精悍、直击要点
- **专业严谨**：学术化、逻辑严密
- **浮夸搞笑**：极度夸张、戏剧化、出其不意

#### 1.4 选择目标字数（可选项，默认2000字）

- **500字**：简短版（2-3分钟）
- **1000字**：标准版（4-5分钟）
- **2000字**：详细版（8-10分钟）
- **3000字**：深度版（12-15分钟）
- **5000字**：超长版（20-25分钟，适合深度专题）
- **10000字**：专著版（40-50分钟，适合系统长文）

#### 1.5 提供素材（可选项）

询问用户是否提供素材（经历、感想、笔记等）。

**方式1：直接粘贴** - 用户直接在对话中发送
**方式2：文件读取** - 从 `user-input/` 目录读取指定文件

### 第二步：智能风格学习

根据主题类别读取示例文章学习风格：

**默认示例文章**：

| 主题 | 默认示例 |
|------|----------|
| 运动健康 | `samples/运动健康 \| 极限健身：为久坐一族编写的健康手册.md` |
| 投资理财 | `samples/投资理财 \| "快、高、稳" 的赚钱骗局又活跃起来了？.md` |
| 生活杂谈 | `samples/生活杂谈 \| 新年的第一次感动.md` |
| 软件开发 | `samples/软件开发 \| 代码审查：从"找茬"到"互助"的心路历程.md` |
| 脱口秀 | `samples/风格深度分析.md` + `styles/浮夸搞笑.md` |

### 第三步：读取模板和风格

1. 读取文章类型模板：`categories/[类型].md`
2. 读取写作风格参考：`styles/[风格].md`

### 第四步：撰写大纲

根据主题、素材、模板、风格，撰写包含以下内容的完整大纲：

- 标题（提供3-5个选项）
- 核心观点
- 主要章节结构
- 关键案例/示例

展示大纲给用户，确认后继续。

### 第五步：撰写正文

严格遵循以下标准：

1. **风格一致性**：应用学习到的风格特征
2. **模板结构**：遵循文章类型模板
3. **字数控制**：按目标字数调整内容深度
4. **素材融合**：优先使用用户提供的真实素材
5. **日期标注**：在文末添加创建日期（YYYY-MM-DD）

### 第六步：保存到 .skill-outputs/blog-writer/

使用 `Write` 工具保存到 `.skill-outputs/blog-writer/` 目录：

**文件名格式**：`[日期]_[主题类别] | [文章标题].md`

示例：`20260131_脱口秀 | 当代职场人的那些迷惑行为.md`

**确认保存**：告知用户保存路径，示例：`✅ 文章已保存到：.skill-outputs/blog-writer/20260131_脱口秀 | 当代职场人的那些迷惑行为.md`

## 扩展已有内容模式（简化流程）

如果用户提供了已有内容（通过 `--input` 参数或直接粘贴）：

### 流程

1. **分析已有内容**：判断类型（想法碎片、草稿、大纲等）
2. **交互式选择**：引导用户选择主题类别、文章类型、风格、字数
3. **整合大纲**：结合已有内容和模板结构
4. **扩展撰写**：保留精华，补充结构，深化内容
5. **保存文件**：保存到 `.skill-outputs/blog-writer/` 目录

### 扩展原则

- **保留精华**：用户的核心观点和原创内容优先保留
- **补充结构**：按模板添加缺失的章节
- **深化内容**：在用户观点基础上增加论证和案例
- **统一风格**：扩展部分与用户原有内容风格一致

## 质量标准

- **原创性**：内容必须原创
- **准确性**：信息、数据准确
- **实用性**：给读者实际收获
- **结构清晰**：标题、开头、正文、结尾完整
