# 简洁明了风格参考

## 风格特点

- 短小精悍，直击要点
- 每句话都有价值
- 快速阅读体验
- 去除冗余表达
- 重点突出明显

## 开头示例

> "Docker容器化部署的5个关键步骤：1. 编写Dockerfile 2. 构建镜像 3. 运行容器 4. 配置网络 5. 数据持久化。下面详细说明。"

## 正文结构示例

### 清晰的标题层级
```markdown
## 一、问题

生产环境内存溢出。

## 二、原因

未设置JVM堆内存参数，使用默认值。

## 三、解决

-Xms2g -Xmx2g -XX:MetaspaceSize=256m
```

### 要点列表化
"Spring Boot自动配置生效的条件：
1. classpath存在指定类
2. 未手动配置Bean
3. 配置文件启用该功能"

### 代码优先
```bash
# 一句话启动项目
java -jar app.jar

# 带参数启动
java -jar app.jar --server.port=8081
```

## 语言特点

### 句式特点
- 短句为主，一句一意
- 省略不必要的连接词
- 直接陈述，少修饰

### 删除冗余表达
| 冗余表达 | 简洁表达 |
|---------|---------|
| 我认为在大多数情况下 | 通常 |
| 我们可以看到很明显的 | 明显可见 |
| 总的来说 | 总之 |
| 也就是说 | 即 |
| 以此类推 | 等等 |

### 关键信息前置
"内存泄漏：检查finalize()方法"
"线程安全：使用ConcurrentHashMap"

## 注意事项

1. **信息密度**: 每句话都要有新信息
2. **可读性**: 短但不能影响理解
3. **完整性**: 简但不能漏关键信息
4. **实用性**: 侧重可操作的步骤
5. **快速扫读**: 支持快速浏览获取要点
