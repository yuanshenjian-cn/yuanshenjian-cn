## 你的任务

全面、深入地分析当前项目，理解其技术栈、架构设计、功能特性和代码质量，并输出详细的分析报告，包含项目待改进的地方。

## 执行流程

### 1. 扫描项目结构

使用 `Glob` 和 `Bash` 工具扫描项目根目录和关键子目录：

- 根目录配置文件（package.json, tsconfig.json, tailwind.config.ts, next.config.* 等）
- 源代码目录结构（src/, app/, components/, lib/ 等）
- 内容目录（content/, posts/ 等）
- 文档文件（README.md, AGENTS.md 等）
- 配置文件（.eslintrc, .prettierrc, .gitignore 等）

### 2. 读取核心文件

使用 `Read` 工具读取关键文件：

- package.json → 了解依赖、脚本、项目元数据
- README.md → 项目概述和目标
- AGENTS.md → 项目规范和开发指南
- 核心配置文件 → 技术栈和工具配置
- 主要源代码文件 → 理解项目架构和实现

### 3. 分析项目维度

从以下维度进行全面分析：

**技术栈分析**：

- 前端框架和库
- 构建工具和编译器
- 样式方案
- 状态管理
- 数据获取方式

**项目架构**：

- 目录结构设计
- 组件组织方式
- 路由结构
- 静态资源管理

**代码质量**：

- TypeScript 使用情况
- 代码规范和风格
- 错误处理模式
- 类型安全性

**功能完整性**：

- 现有功能模块
- 缺失或待实现的功能
- 用户体验考量

**最佳实践**：

- 安全实践
- 性能优化
- 可访问性
- SEO 优化

### 4. 生成分析报告

生成结构化的分析报告，包含以下内容：

```markdown

## 项目概览
- 项目名称
- 技术栈总结
- 项目定位和目标

## 项目结构分析
- 目录结构说明
- 架构设计评估

## 代码质量评估
- 优点
- 待改进点

## 技术实现评估
- 技术选型评价
- 实现方式分析

## 项目待改进建议
### 🔴 高优先级
- [具体问题]：详细描述 + 影响 + 建议方案

### 🟡 中优先级
- [具体问题]：详细描述 + 影响 + 建议方案

### 🟢 低优先级
- [具体问题]：详细描述 + 影响 + 建议方案

## 总体评价
- 项目成熟度
- 发展建议
```

## 规范要求

- 分析要全面、深入，不遗漏重要细节
- 提供具体的改进建议，而非泛泛而谈
