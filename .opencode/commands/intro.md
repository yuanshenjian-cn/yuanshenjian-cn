## 你的任务
全面分析项目结构，理解其类型和核心价值，创建专业的`README.md`文件。如果文档已存在，则智能补充更新。

**重要说明**： 仅在 Commands/Skills 集合项目中，才在 README 中列出"可用命令"和"技能清单"。普通项目不显示这些章节。

## 执行流程

### 1. 扫描项目结构

使用 `Glob` 扫描：
- 根目录文件（配置、入口文件）
- 子目录结构
- 关键文件（package.json, *.md, *.py, *.js 等）
- **特殊目录**：`.**/commands/` 和 `.**/skills/`

**判断是否为 Commands/Skills 项目**：
- 检查是否存在 commands/skills 目录
- 分析项目核心目的（查看 README、主要文件内容）
- 仅在项目主要交付物是 Commands/Skills 时，才生成对应章节

### 2. 识别项目类型

根据文件特征判断：

- **技术代码项目**：package.json/go.mod 等，src/ 源代码目录
- **文档/知识库项目**：大量 Markdown，结构化分类目录
- **工具/配置集合**：配置文件为主，脚本文件
- **模板/脚手架项目**：template/ 目录，初始化脚本
- **资源/素材集合**：图片、音频等素材文件
- **Commands/Skills 集合**：SKILL.md 文件，命令定义文件
- **多用途混合项目**：组合以上多种特征

### 3. 分析项目核心

**读取关键文件**：
- 配置/元数据文件 → 了解基本信息
- 现有 README → 理解已有内容，避免重复
- 代表性文件 → 理解项目内容和价值

**分析维度**：
1. 项目的核心身份和价值
2. 主要内容、功能、资源
3. 目标用户群体
4. 安装、配置、使用方式
5. 独特优势或亮点

### 4. 生成或更新 README

**检测现有 README**：
- 不存在 → 生成全新文档
- 已存在 → 智能分析并补充更新

**智能合并原则**：
- 保留原有项目描述、愿景、背景
- 补充新增功能、模块、文件说明
- 更新目录结构展示
- 完善使用方法和示例

**更新内容识别**：
- 新增功能点 → 添加到功能列表
- 新增 API/接口 → 添加到文档部分
- 新增依赖/配置 → 更新安装说明
- 新增子项目/模块 → 添加到项目结构
- **Commands/Skills 项目特有**：新增 Commands/Skills → 创建对应章节

**README 结构**：

```markdown
# [项目名称]
> [一句话描述核心价值和定位]

## 简介
[项目背景、解决的问题、核心价值]

## 可用命令（仅 Commands/Skills 项目）
| 命令 | 描述 |
|------|------|
| /command-name | 命令描述 |

## 技能清单（仅 Commands/Skills 项目）
### Skill Name
[技能描述和功能列表]

## 内容/功能/特性
[主要模块、功能列表、内容分类]

## 项目结构
```
[目录树]
```

## 使用方法
[安装、配置、使用步骤]

## 贡献指南
[如何参与、提交规范]

## 许可证
[许可证信息]
```

### 5. 保存文件

**目标文件**：`README.md`（或用户指定路径 $ARGUMENTS）

**处理逻辑**：
1. 不存在 → 直接生成保存
2. 已存在 → 智能合并 → 直接覆盖保存

**质量标准**：
- 准确：信息与项目实际一致
- 完整：涵盖核心价值和关键信息
- 清晰：结构合理，易于理解
- 实用：包含用户需要的实际信息
